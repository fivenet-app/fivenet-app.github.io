Nutzen Sie diese Seite, wenn Ihre FiveNet-Einrichtung über die Basisinstallation und die erforderlichen Konfigurationsoptionen hinausgeht.

## Häufige erweiterte Themen

Die häufigsten erweiterten Konfigurationsbereiche sind:

- **Trusted Proxies**, wenn FiveNet hinter NGINX, Traefik, Cloudflare oder einem anderen Reverse-Proxy läuft.
- **Object Storage**, wenn Uploads in S3-kompatiblem Storage statt im lokalen Dateisystem gespeichert werden sollen.
- **OAuth2-Provider**, wenn Benutzer sich über externe Identitätsanbieter authentifizieren sollen.
- **Tracing und Observability** für Debugging, Performance-Analysen und verteiltes Request-Tracing.

Für die wichtigsten Optionsnamen und Konfigurationspfade siehe die [Konfigurationsreferenz](./1.config-reference.md).

### Hinweise zu Tracing

- Setzen Sie `enabled: true` erst, wenn ein funktionierender OTLP-Endpunkt vorhanden ist.
- Verwenden Sie in Produktion einen niedrigen `ratio`-Wert, sofern Sie nicht bewusst mehr Traffic sampeln möchten.
- Halten Sie den Wert von `environment` passend zur Umgebung, in der FiveNet läuft, zum Beispiel `development`, `staging` oder `production`.
- Nutzen Sie `attributes`, um zusätzliche Metadaten anzuhängen, die das Identifizieren von Traces in Ihrer Observability-Plattform erleichtern.

## Reverse-Proxy und Trusted Proxies

Wenn FiveNet hinter einem oder mehreren Reverse-Proxys bereitgestellt wird, prüfen Sie die Option `http.trustedProxies` in Ihrer `config.yaml`.

Das ist besonders wichtig, wenn:

- Client-IP-Adressen korrekt erhalten bleiben sollen
- FiveNet hinter NGINX, Traefik oder Cloudflare läuft
- Sie auf weitergereichte Header aus vertrauenswürdiger Infrastruktur angewiesen sind

Siehe die [Konfigurationsreferenz](./1.config-reference.md#commonly-adjusted-options) für die Einstellung `http.trustedProxies`.

## Object Storage

Standardmäßig speichert FiveNet Uploads im lokalen Dateisystem. Für skalierbareres oder extern verwaltetes Storage können Sie über die `storage`-Konfiguration auf S3-kompatibles Object Storage umstellen.

Das ist besonders nützlich, wenn Sie:

- mehrere Instanzen betreiben
- Cloud-Infrastruktur verwenden
- hochgeladene Dateien außerhalb des Anwendungshosts speichern möchten

Siehe die [Konfigurationsreferenz](./1.config-reference.md#commonly-adjusted-options) für die Einstellung `storage.type` und zugehörige Storage-Optionen.

## OAuth2-Provider

FiveNet kann über die Konfiguration `oauth2.providers` mit externen Authentifizierungsanbietern integriert werden.

Nutzen Sie dies, wenn Benutzer sich über Systeme wie Discord oder einen anderen OAuth2- oder OpenID-Connect-Anbieter anmelden sollen, anstatt nur lokale Authentifizierungsabläufe zu verwenden.

Siehe die [Konfigurationsreferenz](./1.config-reference.md#commonly-adjusted-options) für die Einstellung `oauth2.providers`.

## Tracing

FiveNet unterstützt OpenTelemetry-basiertes Tracing für die Server- und Worker-Komponenten. Damit können Sie Request-Flows, Hintergrundverarbeitung und Performance-Engpässe in komplexeren Umgebungen analysieren.

Tracing wird in der Regel nur benötigt, wenn Sie bereits einen Observability-Stack betreiben oder komplexere Probleme untersuchen.

### Beispielkonfiguration für Tracing

```yaml
tracing:
  enabled: false
  # otlptracegrpc oder otlptracehttp
  type: "otlptracegrpc"
  # OTLP-Tracing-Endpunkt-URL
  url: "https://example.com:4317"
  insecure: false
  timeout: 10s
  environment: "dev"
  ratio: 0.1
  attributes:
    - service.name=fivenet-server
```
