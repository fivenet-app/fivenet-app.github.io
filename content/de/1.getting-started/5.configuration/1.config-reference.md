## Speicherort der Konfigurationsdatei

Standardmäßig sucht FiveNet in dieser Reihenfolge nach einer `config.yaml`:

1. `.` - Das aktuelle Arbeitsverzeichnis
2. `/config` - Häufig bei containerisierten Installationen verwendet

::callout
---
color: secondary
icon: i-mdi-file-document-outline
to: https://github.com/fivenet-app/fivenet/blob/main/config.example.yaml
---
Verwenden Sie die Beispielkonfigurationsdatei als Quelle für Kommentare, Standardwerte und alle verfügbaren Optionen.
::

## Erforderliche Konfigurationsoptionen

Die folgenden Einstellungen sollten für eine funktionierende Installation als Pflichtwerte betrachtet werden:

* `http.publicURL` - Öffentlich zugängliche URL Ihrer FiveNet-Instanz.
* `http.sessions.cookieSecret` - Zufälliges Geheimnis für Session-Cookies.
* `http.sessions.domain` - Öffentlich zugänglicher Domain- oder Subdomain-Name von FiveNet.
* `database.dsn` - MySQL- oder MariaDB-Verbindungszeichenfolge.
* `nats.url` - NATS-Verbindungs-URL, falls nötig inklusive Anmeldeinformationen.
* `jwt.secret` - Geheimnis zum Signieren von JWTs.
* `auth.configAdminGroups` - Gruppen, die in FiveNet Config-Admin-Zugriff erhalten sollen.
* `sync.enabled` + `sync.apiTokens` - Sync-API aktivieren und mindestens einen Token angeben. Dies ist für das FiveNet-Plugin und DBSync erforderlich.

## Hinweise

- Halten Sie `secret`, `jwt.secret`, Session-Secrets und Sync-Tokens nach dem Deployment stabil, außer Sie möchten bestehende Sessions oder Zugriffe bewusst ungültig machen.
- Verwenden Sie die Beispielkonfigurationsdatei für Kommentare und die vollständige Liste der Optionen.
