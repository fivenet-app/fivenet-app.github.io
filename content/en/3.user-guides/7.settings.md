## Features

- Configure job properties such as logo, radio frequency, and Discord bot sync.
- Configure instance-level FiveNet options through the UI (server team).
- For superusers:
  - **Limiter**: Define which permissions each faction can assign to roles.
  - **Laws**: Configure law entries for each faction.
  - **Filestore**: Configure file storage per faction.
  - **App config**: Configure options that affect the entire FiveNet instance.

## Role Management

Factions can manage their own roles when this is allowed by the permission system.

![features-settings-roles.webp](/images/screenshots/features-settings-roles.webp)

## Limiter And Roles Explained

To allow role self-service safely, use the limiter to define the maximum permission set a faction can assign.
For example, you can allow the police faction to see colleague positions on the livemap, while preventing unauthorized factions from assigning that permission.
In practice, the limiter is a list of permissions and attributes available to roles within a faction.

Usually, the server management team defines these limits for each faction.

## Discord Bot - Sync roles to your faction's Discord

Sync examples:

- Employee ranks
- Absent/vacation status
- Server-team groups on Discord servers connected to FiveNet (for example a `Project Leader` role)

## App Config

![Feature Settings - App Config](/images/screenshots/features-settings-appconfig.webp)

App config contains options that affect the whole FiveNet instance, including:

- The default language for the app.
- Default permissions.
- Imprint and Privacy Policy links.
- Livemap sync intervals.
- Banner message with expiration time (for example to announce maintenance).
- And more.
